# Changelog

## 2026-03-14

### Changed

- 重构十六进制输入解析策略，空行始终作为报文分隔符。
- 当输入中没有空行时，改为只在新的一行起始处识别到常见 Ethernet 头时切分新报文。
- 普通输入支持忽略 `0x` / `0X` 前缀，以及行首 `0x0000:` 这类偏移前缀。
- `tcpdump -XX` 输入改为忽略偏移前缀和右侧 ASCII，仅提取十六进制内容。
- 奇数个十六进制字符现在会自动补 `0`，不再因为格式不完整而中断导出。
- 不完整或协议字段不合法的报文片段现在也会直接写入 `pcap`。

### Removed

- 移除了主界面的错误列表区域。
- 移除了解析失败列表、`PacketParseError` 模型以及相关 UI 绑定。
- 移除了“成功 N 个，失败 M 个”的结果提示，改为只显示实际导出的报文数量。

### Fixed

- 修正了 `0x0000:` 这类地址偏移前缀被误当成报文内容的问题。
- 修正了 `tcpdump` 偏移跳变或非零起始时无法导出的情况。

### Verification

- 通过 `build.ps1 -Configuration Release -StopRunningApp` 完成构建与测试验证。
- 当前控制台测试共 13 项，全部通过。
