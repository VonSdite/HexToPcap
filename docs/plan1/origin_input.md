我要实现一个windows应用程序，它的功能比较简单，它叫HexToPcap，它的主要功能就是将用户传入的十六进制字符串转成报文pcap文件，一次性用户可能传入多行十六进制字符串，一个报文可能连续多行，但一个报文不会被空行分隔开，多个报文连续多行或者夹杂空行分割报文；主题是一个textarea，用户直接粘贴到textarea后，进行转换；允许用户配置转换后pcap文件存放的位置，pcap文件命名为"年月日时分秒-报文个数.pcap",转换后如果它发现能关联上windows wireshark程序，则提示是否使用wireshark打开；在配置管理中看起来还需要支持配置wireshark程序的路径，如果没配置，它在转换pcap后自己判断能不能关联上，如果配置了，就是用配置的wireshark提醒打开；hex文本有时候可能是使用tcpdump -vv -nn -XX方式抓包来的，它会带着hex地址，hex文本，ascii转义后的字符串，这种格式我也希望贴进来能支持自动转成pcap。
